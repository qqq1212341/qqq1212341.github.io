단계구분담당내용
1컨셉·PRD 자동 game-concept PRD/README(SoT) 정의 + 8렌즈/3게이트(game-concept). 발산·스펙 문답은 superpowers(brainstorming→writing-plans), 포화도 근거는 marketing-skills(competitor-profiling·customer-research). 판정은 game-concept가 유지.
2엔진·밸런스 자동 balance_sim 코어 루프 구현·플레이가능 + ★제작 기본값 3종을 코어에 함께 심는다(친절 FTUE·주스 표준·로직/표시 분리 — game-feel-playbook). 정답 있는 로직(밸런스·규칙)은 superpowers TDD, 재현 안 되는 버그는 systematic-debugging.
3제작 폴리시 자동 codex-image 아트 생성(codex-image 게임 에셋 모드 — 자원 아이콘·업그레이드 아이콘·히어로·ui-panel까지 1회차) · 사운드 · 셸 게임화(계기판·게임카드·소켓·풀블리드 로비) + UX 라이팅 사전 적용 + 눈검증(접지/스케일/일관성) 통과 (하위 최저 상태). 상세=game-feel-playbook. 스크린샷 안 보면 done 아님.
4수익엔진·추적 자동 store-deploy AdMob 통합(스플래시→전면광고→메인 + 보상형 + 배너, 매출 70%) + 트래킹 SDK 심기(analytics.js 드롭인 = GA4 무료, 세션·진행·이탈·수익·리텐션 이벤트 → 그로스 진단 연료). 둘 다 공장 기본. 무수익/무추적은 컨셉서 명시한 경우만.
5QA 자동 qa-harness 눈검증(shoot→루브릭→qa/visual-qa.json) + Playwright 회귀 스위트. 화면이 루브릭 통과해야 done(거짓 100% 차단).
6빌드·스토어 자동 store-deploy 서명 빌드 · 스토어 자산(스샷/피처) · 카피 한/영 (하위 최저 상태)
7콘솔 등록 사람 게이트 store-deploy 폼·업로드=자동, 로그인/2FA=사람
8심사·출시 사람 게이트 사람 최종 제출·리젝 대응 + 웹 라이브 / 스토어 게시·결제 (하위 최저 상태)
9그로스·획득 자동 game-grow 0원 마케팅(숏폼·커뮤니티·크로스프로모션) + 리텐션(푸시·일일) + Google Ads 자동생산. 광고비 집행만 사람 게이트.
